## 🎓 ТРЕНЕР МОД — Навчання Бота

### **Призначення**
Навчати бота новим знанням через звичайні розмови.

### **Як користуватися:**

1. **Увімкни тренер мод**
   ```
   /admin → 🎓 Тренер мод
   ```

2. **Побачиш підтвердження:**
   ```
   🎓 ТРЕНЕР МОД АКТИВНИЙ
   
   Просто пиши — я запам'ятаю нові факти.
   Кидай фото — збережу в каталог.
   Коли закінчиш — натисни "Завершити"
   
   [🔚 Завершити тренування]
   ```

3. **Розказуй нові факти:**
   ```
   "Плетіння Картьє коштує на 200 грн дорожче ніж Бісмарк"
   "Для кільця розмір 18 оптимальна маса 4-5 грам"
   "Застібки-карабін найнадійніші для браслетів"
   ```

4. **Додавай фото:**
   - Готові вироби на вагах
   - Приклади плетінь  
   - Процес роботи

5. **Завершуй тренування:**
   ```
   Натисни [🔚 Завершити тренування]
   ```

### **Що запам'ятовує:**
- ✅ Нові ціни та розрахунки
- ✅ Особливості плетінь та виробів
- ✅ Відповіді на часті питання
- ✅ Фото-приклади для каталогу
- ✅ Терміни та процеси роботи

## 📋 БАЗА ЗНАНЬ — Перегляд та Редагування

### **Призначення**
Переглядати всі знання бота та керувати ними.

### **Структура записів:**
```json
{
  "title": "Питання або тема",
  "content": [
    {"text": "Детальна відповідь бота"}
  ],
  "status": "confirmed"
}
```

### **Як користуватися:**
1. `/admin` → **📋 База знань**
2. Побачиш список всіх записів (по 5 на сторінку)
3. Можеш переглядати, редагувати, видаляти

### **Поточний стан v3:**
- **15 записів** перенесено з v2
- Всі підтверджені та активні
- Інтегровано в промпт консультанта

## ⚠️ НЕПІДТВЕРДЖЕНІ ЗАПИСИ

### **Що це?**
Знання які бот дізнався, але ще не використовує в консультаціях.

### **Як з'являються:**
- Автоматичний аналіз розмов з клієнтами  
- Виявлення нових питань без відповідей
- Експорт з логів тренувань

### **Як підтверджувати:**
1. `/admin` → **⚠️ Непідтверджені (X)**
2. Переглянути кожен запис
3. Натиснути **✅ Підтвердити** або **❌ Відхилити**
4. Підтверджені записи стають активними для консультацій

### **Рекомендації:**
- Перевіряй щотижня чи є нові записи
- Підтверджуй тільки точну та актуальну інформацію
- Відхиляй застарілі або неправильні дані

## 🔧 СИСТЕМА МОНІТОРИНГУ

### **Health Monitor v3 (AUTO-RESTART + ALERTS)**
Автоматичний моніторинг стану бота з перезапуском при проблемах.

**Встановлення:**
```bash
sudo bash tools/install_monitor.sh
```

**Що відслідковується:**
- ✅ **Systemd сервіс** — чи процес активний?
- ✅ **Telegram API** — чи бот відповідає на `/getMe`?
- ✅ **Auto-restart** — перезапуск через `systemctl`
- ✅ **Telegram алерт** — повідомлення при проблемах

**Приклад алерту:**
```
🚨 InSilver бот — проблема

⏰ Час: 2026-03-25 03:45:12  
⚠️ Причина: Telegram API не відповідає
🔁 Рестартів всього: 2
🔄 Дія: ✅ перезапущено systemd
```

### **Bot Manager (MANUAL CONTROL)**
Зручні команди для ручного керування ботом:

```bash
python3 tools/bot_manager.py status    # детальний статус
python3 tools/bot_manager.py restart   # перезапуск
python3 tools/bot_manager.py logs      # останні 30 рядків логів  
python3 tools/bot_manager.py follow    # live логи
python3 tools/bot_manager.py health    # швидкий health check
```

**Приклад виводу:**
```
✅ insilver-v3 — активний
📍 PID        : 2433
🧠 RAM        : 45.2 MB
🔁 Рестарти : 0  
⏰ Останній старт: Tue 2026-03-24 20:18:11
```

### **Конфігурація алертів:**
**Для Влада (власника):**
```bash
# Додати в .env
ALERT_CHAT_ID=123456789  # chat_id Влада
```

**Для розробника:**
```bash
# Алерти надсилаються Сашку за замовчуванням
# Або через systemd env
sudo systemctl edit insilver-monitor
```

### **Логи моніторингу:**
```bash
# Live логи monitor системи
journalctl -u insilver-monitor -f

# Файловий лог
tail -f logs/monitor.log
```
